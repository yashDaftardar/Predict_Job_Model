Position:

Software Test engineer for applications and systems software as a member of a software engineering Integrated Product Team (IPT). Will be responsible for supporting, planning, coordinating, and executing verification, validation and testing of application and systems software. Incumbent must be able to qualify as a DoD Cybersecurity Workforce member and maintain a SECRET clearance.

Duties:

-Work closely with software developers, engineers, business analysts, and requirements personnel to plan, coordinate and execute static analysis and dynamic testing of systems/software code to verify, validate acceptance and technical specifications.

-Evaluate the test basis and support the refinement of requirements to ensure testability, clarity and correctness.

-Author and maintain software test cases and procedures in accordance with organizational policies and procedures.

-Conduct test design activities to identify and create both manual and automated test cases, test procedures and test scripts traceable to system requirements.

-Assist in the creation of software test plans to document the overall test strategy, test approach, test process, test schedule, pass/fail criteria, test environment architecture, test lab resources.

-Conduct static and dynamic testing at all software testing levels to include integration, functional, system, and acceptance.

-Implement and apply automated testing tools to support planning, test execution, regression testing, defect tracking and configuration management.

-Work closely with stakeholders, sponsors, and customers to determine acceptance criteria and coordinate certification test events with end users.

Knowledge, Skills and Abilities:

1. Knowledge of testing business application and enterprise information systems.

2. Knowledge of static and dynamic software test activities.

3. Knowledge of test automation and using automated test tools

4. Knowledge of waterfall, agile, or incremental software development life cycle models.

5. Knowledge of computer operating systems and database management systems/software.

6. Skilled in utilizing software development management and issue tracking tools (JIRA).

7. Skilled in creating software test plans, test cases, test procedures, test scripts, and test data conditions.

8. Skilled in utilizing testing and application life cycle management tools (e.g. Micro Focus ALM) to build test suites/sets, capture test data, and analyze results.

9. Skilled at mapping test cases to requirements and providing input to generate a requirements traceability matrix.

10. Skilled in automating test cases using test automation tools.

11. Ability to test computer operating systems, database management systems, and application software integrated on physical and virtual servers and clients.

12. Ability to assess software products for readiness and determine pass/fail criteria.

13. Ability to determine test approaches, develop test schedules, and identify risks associated with testing.

14. Ability to capture test data, analyze test results and clearly communicate (orally and written) findings and/or deficiencies.

15. Ability to coordinate and collaborate with members of a software development or systems engineering team to accomplish project team goals and objectives.

16. Ability to conduct software/system integration, installation, configuration, functional, performance, regression and system recovery testing.
