Software Integration Engineer

AVIATION SYSTEMS ENGINEERING CO., INC
Washington, DC
Description:

ASEC is currently searching for a Software Integration Engineer to support the Program Office in Washington, DC. The Software Integration Engineer will be responsible for working with the System/Software and Algorithm teams and performing software integration activities. The candidate will be responsible for defining the continuous integration toolset, strategies for Algorithm and software integration, and release activities. They will also be involved in defining interfaces at various levels to perform optimal integration test activities and acceptance testing. As a member of the integration team, they will be working in an Agile DevSecOps environment, supporting design, implementation, and testing efforts to ensure successful integration of software, hardware, and other requirements across the total engineering effort.

. Requirements:


Education and Experience:

  • Bachelors degree in Software Engineering, Electrical Engineering, Computer Science or similar technical degree.
  • Five (5) years of professional software integration experience.
  • Experience working with Python, Red Hat Enterprise Linux (RHEL) 7 operating system, and Container (Docker, Kubernetes, etc.).
  • CompTIA security +/CISSP

Required Clearance: This position is subject to a government security investigation and must meet eligibility requirements for access to classified information. Requires an ACTIVE SECURITY CLEARANCE.

Skills and Qualifications:

  • Knowledge of OS operation in a cross-platform environment (Linux/Windows).
  • Excellent verbal and written communication skills.
  • Excellent interpersonal and customer service skills.
  • Detail oriented.
  • Ability to organize and prioritize tasks.
  • Demonstrated critical thinking and problem-solving skills.
  • Proficient with Microsoft Office Products or similar software.
  • Experience utilizing software tools for engineering and programming languages.
  • Ability to adapt to evolving technology.
  • Ability to work as part of a team.

Tasks/Responsibilities:

  • Owner of the software configuration management strategy.
  • Define software integration/release at various levels with clear reuse strategy.
  • Define software continuous integration/development mechanism.
  • Drive and coordinate with teams to follow life-cycle development and deployment procedures.
  • Define integration test strategy and overall product performance.
  • Support the continual certification and accreditation (C&A) of the DevSecOps environment, continuous ATO process.
  • Define and document inheritable controls available via the pipeline that will be incorporated in software packages as required.
  • Provide ongoing and continual operations and maintenance support in the following areas:

o Assist in the maintenance and upgrades of DevSecOps toll chain and pipeline components including a code repository, and artifact repository, security scanning, testing tolls and an orchestrated integration and delivery platform which enables automated application building, testing, securing and deployment.

o Document any changes to the architecture for proper configuration management, detail each functional component, and document dependencies with other systems.

o Support the deployment of the platform environment in both integration and production environments.

o Provide documentation on the functions of the pipeline, artifacts, and hardened containers available via the pipeline, container repository and how application developers can incorporate the pipeline into their development activities.

o Record, tract and report integration performance and software quality metrics.

  • Collaborate with other systems integrators, test engineers, performance engineering, systems engineering, software engineering, and management to work within an Agile DevSecOps environment to ensure a high quality, stable, and functional system.
  • Support performance and quality efforts.
  • Identify, document, report, and track issues with the software, hardware and related systems and perform root cause analysis of issues.
  • Capture debug and diagnostic information for identified issues to facilitate software and systems engineering resolution of issues.
  • Drive all integration, application, and systems issues to closure.
  • Identify and communicate product and project risks and opportunities.

Other Requirements:

  • Requires occasional travel.
  • Requires flexibility in work hours.
