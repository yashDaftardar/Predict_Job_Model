Full Job Description

Impact:
  • Build software applications that are user-facing, integrating deeply with our data platform as well as traditional transactional systems that support application behavior.
  • Leverage version control while developing features in order to define version compatibility, review status, and association with User Stories or tickets.
  • Write idiomatic code, following best practices whenever possible.
  • Participate in peer review both as a reviewer and a receiver of constructive feedback.
  • Hold our engineering team to a high standard of authorship.
  • Fix bugs! Address technical debt! Improve the quality of our code base as we iterate on the application stack that already exists, both during bug fixes and feature development.
  • Work with a variety of programming languages and environments, ranging from scripting to compiled languages. Familiar with object oriented, functional, and structured programming patterns.
  • Develop software around database technologies varying from horizontally scalable eventually consistent systems, to vertically scalable transactional systems.

Background:
  • 3+ years experience developing in a strongly typed, object oriented, server-side language such as Kotlin, Java, or C#.
  • Experience leveraging a server-side web framework such as Spring MVC, .NET Core MVC, Rails, Django, etc.
  • Experience with a server-side build system such as Maven, Crate, or Gradle.
  • Experience with one or more server-side testing frameworks such as JUnit, Mockito, and NUnit.
  • Experience with producing and consuming RESTful web API endpoints.
  • Familiarity with modern API development tools such as Swagger and Postman.
  • An understanding of common design and object orientated patterns.
  • Familiarity with NoSQL and transactional database management systems ranging from Elasticsearch or Cassandra to PostgreSQL or MSSQL.
Nice to have:
  • Familiarity with the NodeJS ecosystem and associated tooling such as NPM, Yarn, Webpack, and Babel.
  • Familiarity or interest in alternate concurrency paradigms such as message queues or pub sub.
Absolute must:
  • Comfortable learning new languages, technologies, and methodologies quickly.

The above statements describe the general nature and level of work being performed in this job function. They're not intended to be an exhaustive list of all duties, and indeed additional responsibilities may be assigned by Deep 6.

At Deep 6, we appreciate the opportunity to benefit from the diverse backgrounds and experiences of others. Because of our deep commitment to respect every individual, Deep 6 is an equal opportunity employer.
