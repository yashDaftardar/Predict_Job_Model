The Software Engineer is responsible for the development of product / platform enhancements, code review of 3rd party vendor applications, the day-to-day maintenance of the application systems in operation, including tasks related to identifying and troubleshooting application issues and issues resolution or escalation. Responsibilities also include root cause analysis, management communication and client relationship management in partnership with Infrastructure Service Support team members. Ensures all application changes are made in accordance with life-cycle methodology and risk guidelines.

Day to day activities include:
Represents the Application Development team on all active projects
Runs the daily application development stand-up meetings
Monitors process and software changes that could application development procedures
Measures application development key performance indicators, for example: bug density, sprint analysis, test coverage, etc.
Follows established procedures for change management across applications where changes are necessary
Creates WIKI articles for common processes that are repeated and can be shared with others.
Work with the Product Team to implement new application enhancements and features
Develop product unit tests that will be used for future application regression testing. Conduct unit testing and provide report out to development manager.
Conduct risk-based code reviews for applications
Provide technical and architectural guidance to our development partners
Design and estimate new platform changes or enhancements
Suggest fixes to complex issues by doing a detailed analysis of root cause and impact of the defect.
Coordinate with Application Development Team to successfully deploy software releases in both User Acceptance Testing and Production environments.
Setup environments for all new implementation
Configure Single Sign On integrations for client builds
